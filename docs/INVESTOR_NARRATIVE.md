# Aegis — Investor narrative (one-pager)

**Brand:** Aegis Trust OS  
**Tagline:** Trust infrastructure for high-stakes human services.  
**Secondary:** AI assists. Humans adjudicate.

## Problem
High-stakes human services (executive concierge, security-adjacent staffing, private care) cannot be trusted as open gig marketplaces. Buyers need **curated assurance**; providers need **credible demand**. Generic platforms optimise listings. Horizontal IDV vendors optimise fintech onboarding. Neither owns **role-specific clearance + UK compliance evidence + placement orchestration**.

## Solution
**Aegis** — hybrid agency + vertical trust-tech:

1. **Identity** — pluggable IDV (mock now; Trulioo/Socure-class in production) + credential wallet + audit  
2. **Compliance** — UK-shaped matrices (RTW, DBS, SIA, safeguarding) with **human** adjudication  
3. **Orchestration** — structured requests, explainable matching, NDA/contracts, bookings, incidents, KPIs  

We **sit on IDV rails**. We **own the vertical layer**. We do **not** attempt to out-build global document coverage or fraud ML.

## Moat
Verification + adjudication + category ops + evidence trails — not listings. Switching costs rise with wallets, audit history, partner workflows, and proven incident discipline.

## Wedge
**London premium non-clinical concierge** (PA, chauffeur, travel, stylist, beauty, hospitality) → security-adjacent (SIA) → care only after deep ops tooling. Start where regulation is lighter and willingness to pay is higher.

## Model
Tech product + white-glove Trust & Safety. Marketplace network effects after standards proven. Premium take rates; **Aegis for Agencies** white-label path.

## Traction (software MVP live)
- Runnable trust engine + concierge orchestration  
- Seeded London demo, ops console, live KPI dashboard  
- Security/care as honest scaffolds (waitlist + matrix)  
- Competitive posture and demand-validation scripts documented  

## Scenarios (guarded)
| Case | Outcome |
|---|---|
| **Optimistic** | London pilot + enterprise white-label; strong trust metrics |
| **Base** | Niche profitable player in concierge/security |
| **Pessimistic** | Cold-start stalls — mitigate via interviews + agency partnerships first |

## Pilot KPIs
| KPI | Target |
|---|---|
| Median vetting time | &lt; 48h |
| Booking success | &gt; 70% |
| Repeat engagement | &gt; 50–60% |
| Incident rate | &lt; 0.5–1 / 1,000 bookings |
| NPS (proxy / survey) | &gt; 70 |
| Active vetted Tier-1 providers (12 mo) | 100+ |

## Why now
IDV vendor maturity (buy the base layer), UHNW/executive demand for vetted lifestyle ops, and regulatory pressure against shallow marketplaces in care/security.

## Ask framing
Fund **vertical trust OS** execution and London pilot — not a horizontal IDV war. Raise against demand validation + pilot trust metrics. Make Aegis the standard orchestration layer above identity infrastructure.
